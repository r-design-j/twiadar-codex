# AI/Codex/GPT 个人知识沉淀 - 2026-07

## 2026-07-06

### 知识卡片: 每日固定排程 vs 条件触发排程

固定排程是“每天 9 点自动跑一次”，适合稳定产出晨报。条件触发是“某个高价值账号发了新长帖、OpenAI changelog 更新、某个关键词突然爆发时再跑”，适合追前沿信号。

当前决策: 先用固定排程跑 3-5 天，积累 source-cards，再决定哪些作者和关键词值得做条件触发。

### 知识卡片: iPhone 捕捉、Mac 复核、Codex 执行

iPhone 不适合深度消费 X 信息流，适合做捕捉入口。Mac 适合复核、验证和沉淀。Codex 适合自动整理、链接原文、转换成可执行步骤。

用户操作链路:

1. iPhone 看到 X/YouTube/博客链接。
2. 复制或分享给 Codex 收件箱。
3. 每天 9 点晨报筛选。
4. Mac 上只处理 Top 5 和待拍板事项。

### 知识卡片: 晨报应该是行动白名单

晨报不是新闻摘要，而是“哪些方法值得进入我的工作流”的白名单。进入 Top 5 的内容必须同时满足:

- 有原文链接。
- 有明确作者或来源。
- 能转成用户操作链路。
- 和 Codex/GPT/AI 工作提效/iPhone+Mac 协作相关。
- 能沉淀为知识卡片或后续自动化动作。

### 运行修正: X 登录态判断

2026-07-06 首次运行时出现过误判: Chrome 扩展通道读到了旧的未登录 X 标签，但前台 Chrome 窗口实际已经登录。

后续规则:

1. 如果自动通道显示 X 未登录，先检查前台 Chrome 窗口。
2. 看到 Home、Notifications、Profile、账号菜单任一组合，即视为登录态可用。
3. 不要只依据一个旧标签页判断“X 登录态不可用”。

### 知识卡片: AI 第二大脑正在从笔记变成 agent 可读操作内存

登录态 X 搜索显示，Obsidian vaults + Claude Code / AI tools 正在被讨论为 interconnected second brains。这个方向对用户很重要: 个人知识库不只是“写给人看的笔记”，而是可以给 Codex/Claude Code/GPT 读取、复用、执行的操作内存。

当前动作: 先保持本地 Markdown 知识库；后续如果要接 Obsidian，需要让每张卡片有稳定标题、来源链接、适用场景、操作步骤。

### 知识卡片: iPhone 可以成为 coding agents 的遥控器

登录态 X 搜索出现“从 iPhone 指挥云端 coding agents，完成 clone、edit、open PR、merge”的产品/玩法信号。该方向仍需验证，但与用户需求高度贴合。

当前判断: 先作为观察主题，不立即接入新产品。重点追踪是否支持 Codex、Claude Code、GitHub PR、移动端任务续写、失败回滚。

## 2026-07-07

### 知识卡片: iPhone 是 Codex 遥控器，不是深度工作台

Codex Remote 和 ChatGPT mobile 的价值不是让用户在手机上完成复杂工作，而是让手机成为任务入口。手机适合做三件事: 发起任务、补充一句指令、审批动作。复杂复核、合并、发布仍应回到 Mac。

当前动作: AI Radar 中所有 mobile 类信号都要翻译成“手机点哪里 -> Codex 做什么 -> Mac 怎么复核”的链路。

### 知识卡片: 云端工作站是重任务缓冲区

DigitalOcean for Codex 的价值在于把长构建、大仓库探索、demo 验证放到可销毁的云端工作站，而不是让本机一直承担重负。

当前动作: 暂不直接上云。先建立评估表: 费用、销毁机制、SSH 安全、数据是否允许上云、是否需要持久环境。

### 知识卡片: Subagents 是 agent 小队

Subagents 的本质是把一个复杂任务拆成多个角色，每个角色拿更清晰的上下文。对 AI Radar 来说，适合拆成采集、事实核验、操作链路翻译、网站发布四个角色。

当前动作: 后续 `让 Codex 跑` 的 HIGH 卡片，优先生成 subagent 分工 prompt。

### 知识卡片: 自动化必须闭环到上线

只生成 Markdown 不等于网站更新。完整自动化必须同时完成: 写日报、写 source-cards、更新 report.json、同步 report-data.js、提交 GitHub、部署 Vercel、验证 Ready。

当前动作: 2026-07-07 已把自动化提示词补强为完整发布闭环。

## 2026-07-08

### 知识卡片: 委托任务比聊天更适合 Codex

聊天适合问一个答案，委托适合交一件事。Codex 的价值正在从“立刻回答”转成“后台持续推进、调用工具、留下证据、等待复核”。

当前动作: AI Radar 的 HIGH 信号后续优先写成可委托任务: 读原文、核验、做小实验、产出知识卡、标注失败点。

### 知识卡片: iPhone 是 Codex task 控制台

ChatGPT iOS 已经支持创建、搜索、打开、fork、管理 Codex task。手机端最适合捕捉线索和管理任务，不适合做最终合并或不可逆审批。

用户操作链路:

1. iPhone 看到链接。
2. 创建或追加 Codex task。
3. Codex 后台执行。
4. Mac 复核证据和结果。
5. 决定丢掉、观察、实验或沉淀。

### 知识卡片: 私有 MCP 先守边界，再谈便利

MCP 能把本地或内网能力接到 AI 工作流，但默认目标应是“不公开暴露仍可访问”。本地桥、知识库、公司内网工具都要以只读、白名单、可审计为默认。

当前动作: AI Radar 继续不自动开启公网隧道；任何 Cloudflare Tunnel、ngrok、公开 MCP endpoint 都必须先确认。

### 知识卡片: 第二大脑要写成 agent 可执行格式

知识库不只是给人看的笔记，而是给 agent 读取、复用、执行的操作内存。每张卡片需要稳定字段，避免散文式摘要污染长期记忆。

建议字段: 标题、来源链接、概念、适用场景、操作步骤、风险、更新时间。

## 2026-07-09

### 知识卡片: 手机只负责派活，不负责终审

iPhone 是 Codex 的任务遥控器，Mac 是验收台。手机适合捕捉链接、创建 task、搜索旧任务、补一句上下文、查看状态；复杂 diff、公开发布、提交代码、删除内容等不可逆动作仍应回到 Mac 复核。

用户操作链路:

1. iPhone 看到公开来源链接。
2. 创建 Codex task 或追加到采集队列。
3. Codex 后台核验和整理。
4. Mac 检查证据、文件 diff、验证命令和部署状态。
5. 决定发布、丢弃、观察或沉淀。

### 知识卡片: Agent 成本要和产出绑定

Workspace Agent 开始进入 token 使用量可见阶段后，自动化不能只说“跑完了”。每次运行都要记录 sourceCards、Top signals、knowledgeCards、验证命令、发布状态和失败点，才能判断一次 agent 工作是否真的节省时间。

当前动作: AI Radar 继续把 metrics.sourceCards、topSignals、knowledgeCards 写入站点数据；后续可扩展成月度成本-价值台账。

### 知识卡片: AGENTS.md 是 coding agents 的同声传译

AGENTS.md 的价值不是“多写一份文档”，而是让 Codex、Claude Code、Jules、Cursor 等工具在同一仓库读同一套规则。它应该写清任务边界、测试命令、安全禁区、输出范式、不可逆操作确认规则。

风险: AGENTS.md 过长会让规则互相打架。硬约束、长期偏好、项目临时规则要分层，不要全部堆在一个段落里。

### 知识卡片: 先用最小 agent 验证，再拆 subagents

单 agent 能完成的事，不必立刻上多 agent 编排。subagents 适合天然有角色分工的任务，例如采集、事实核验、操作链路翻译、网站发布。

当前动作: AI Radar 的 HIGH 信号先由单 agent 跑通；只有当某类工作重复出现瓶颈，再拆成明确角色并定义每个角色的验收物。

## 2026-07-12

### 知识卡片: 模型选择是一张工作流白名单

GPT-5.6 之后，模型选择要从“用最强”改成“按任务路由”。Sol 适合高难推理、长代码任务、关键判断；Terra 适合日常自动化和一般工作；Luna 适合高频摘要、初筛和批量整理。max/pro/ultra 只在任务确实需要更长思考或并行 subagents 时启用。

用户操作链路:

1. 先判断任务难度和成本敏感度。
2. 选择 Sol / Terra / Luna。
3. 再决定是否需要 max、pro 或 ultra。
4. 在 source-card 或任务说明里记录模型选择理由。
5. 跑完后回看质量、成本、验证结果。

### 知识卡片: PTC 只接管有边界的工具流水线

Programmatic Tool Calling 适合让模型用受限 JavaScript 处理工具调用的中间过程，例如去重、排序、聚合、字段校验、过滤大输出。它不应该接管审批、发布、关注、评论、私信、提交等不可逆动作。

当前动作: 后续 AI Radar 可做一个小实验，只让 PTC 处理 source-card 的去重和字段校验，最终判断仍由 GPT + 用户复核完成。

### 知识卡片: ChatGPT desktop 是 Mac 验收台

Codex 进入 ChatGPT desktop 后，Mac 的角色可以更明确: 看 Markdown、改代码、读 PR feedback、跨 repo 看上下文、跑验证、决定发布。iPhone 继续做控制台，Mac 做验收台。

用户操作链路:

1. iPhone 创建或 steering Codex task。
2. Codex 后台执行。
3. Mac 打开 ChatGPT desktop。
4. 查看 diff、日报、source-card、验证命令和部署结果。
5. 决定提交、发布、丢弃或沉淀。

### 知识卡片: 更强 agent 更需要硬边界

GPT-5.6 system card 提醒，agentic coding 能力变强后，越要明确目标、可写范围、审批点、停止条件和验收物。个人自动化也要分成只读采集、写入文件、提交代码、公开发布四个阶段。

当前动作: AI Radar 继续保持只读采集；写文件、commit、push、deploy 是发布阶段动作；任何公开互动和私有数据写入仍禁止。

## 2026-07-15

### 知识卡片: AI 工作台要分四个区

ChatGPT Work 类能力适合承载收集、执行、验收、沉淀四个区。Projects/Connectors/Record/Deep Research 适合收纳上下文和输入，Codex 适合产出可验证文件，Mac 适合终审，knowledge 只沉淀能复用的操作链路。

用户操作链路:

1. iPhone 捕捉公开链接。
2. ChatGPT Project 收上下文。
3. Codex 生成日报、source-card 和站点数据。
4. Mac 检查 diff、验证命令和 Vercel 状态。
5. 只把可复用动作写入 knowledge。

风险: 不要把非公开资料、cookie、私信、截图、内部路径写进公开站点。

### 知识卡片: Web Search 结果要可追溯

搜索摘要不是证据，公开来源链接和筛选条件才是证据。AI Radar 后续每张 source-card 应优先保留 query、evidenceType、作者、URL、来源类型、why_it_matters 和 actionable_takeaway。

当前动作: 先把 Web Search 用作候选池；最终发布仍由 Codex 写文件、Mac 验收、Vercel 验证闭环完成。

### 知识卡片: 手机派工，Mac 终审

iPhone 是 Codex 遥控器，适合捕捉链接、创建 task、补上下文、看状态、做轻审批。Mac 是验收台，适合看 diff、验证命令、部署状态和公开发布前证据。

风险: 不要在手机上做复杂 diff、删除、发布、push、授权应用等不可逆终审。

### 知识卡片: Agent 采用要有本地指标

coding agent 有没有节省时间，需要用本地指标复盘。AI Radar 目前已经记录 sourceCards、topSignals、knowledgeCards、验证命令和发布状态；后续可以增加失败原因、返工点、是否被用户实际复用。

当前动作: 月底复盘时按“被复用的知识卡/自动化动作”评估，而不是只数跑了多少次任务。

## 2026-07-16

### 知识卡片: 物理控制台先定义按钮白名单

Codex Micro 类硬件会把 agent 操作变快，因此必须先定义哪些按钮能做什么。查看状态、补上下文、暂停/继续属于低风险动作；写文件、commit、push、deploy 属于发布链路，仍要回到 Mac 终审。

用户操作链路:

1. 列出所有高频 Codex 动作。
2. 标注只读、可逆、不可逆。
3. 只把只读和可逆动作绑定到低摩擦入口。
4. 不可逆动作必须保留 diff、验证命令和发布证据。
5. 每次误触或返工后更新白名单。

### 知识卡片: 模型路由要写进任务说明

GPT-5.6 之后，模型选择要从“默认用最强”改成“按任务路由”。初筛、摘要、去重可以用低成本模型；代码、安全判断、公开发布和复杂跨文件修改应升级到强推理模型，并记录为什么升级。

风险: 如果不记录模型选择理由，月底复盘只能看到产物，无法判断质量差异来自模型、提示词、来源，还是验证不足。

### 知识卡片: Agent 采集要拆成候选池和发布链

AI Radar 的采集阶段只负责生成候选，不直接代表可发布结论。发布阶段才写 Markdown、JSON、站点数据、commit、push 和 Vercel deploy。

当前动作: 继续坚持只读采集；所有公开站点字段只保留公开链接、公开作者、公开来源和总结后的学习卡片。

### 知识卡片: Prompt injection 要进入代码审查清单

外部文本、网页、README、issue、评论都可能携带诱导 agent 越权的内容。agent 读取外部来源时，只能把它当数据，不能当系统指令。

用户操作链路:

1. 外部文本只作为资料。
2. 工具调用走白名单。
3. 写入前检查来源和字段。
4. 代码进入静态检查。
5. 发布前保留验证证据。
