3. 多 agent 分工

Subagents 的价值是降低一个 agent 同时承担太多角色的混乱度。对于 AI Radar，可以拆成:

- source scout: 找来源。
- fact checker: 回查官方文档和一手链接。
- workflow translator: 翻译成“点哪里、看到什么、再做什么”。
- publisher: 更新 JSON、推 GitHub、部署 Vercel。

## 白话版

今天这版可以理解成: Codex 不是一个坐在电脑里的“写代码小助手”了，它越来越像一个可以被你调度的工作团队。

iPhone 是遥控器。你不用在手机上把每条 X 都读完，手机只负责把“这个可能有用”丢进篮子里。真正判断值不值得学、要不要沉淀，交给每天 09:00 的 AI Radar。

DigitalOcean 云端工作站像临时外包工位。你的 Mac 不一定要背所有重活，长时间构建、大 repo 探索、demo 验证，可以考虑交给云端机器。

Subagents 像一个小队。一个人同时负责找资料、验真假、写总结、发网站，容易乱；拆成几个角色后，每个人只干一件事，结果更稳。

自动化像印报纸的完整流水线。之前的问题是: 报纸内容写过一次，但没有每天确认“重新排版、印刷、送到门口”。现在已经补成闭环: 生成、更新网站数据、推 GitHub、部署 Vercel。
